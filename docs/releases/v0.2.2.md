# v0.2.2

## Changes
- Fixed direct UDP-over-stream relay cancellation so burst writes, partial client frames, pinned first targets, and delayed target replies keep flowing instead of being dropped or retargeted.
- Made Linux UDP GSO opt-in for release safety: `ZUICITY_ENABLE_GSO=1` enables it, while the default path stays plain and handshake-safe on GSO-hostile veth/tun/VPN/virtio/cloud egresses.
- Tightened server proxy-loop classification so empty pre-header peer closes are quiet, partial headers remain counted, target open failures move to debug `targetStream`, and exact Quinn `connection lost` closes are treated as clean peer closes.
- Bounded the high-pressure wedged-relay regression with an explicit short QUIC idle timeout, while preserving the dropped-client leak scenario.

## Validation and benchmark notes
- Local release gates passed: format, workspace check/test excluding benchmarks, focused transport/server tests, packaging validation, package smoke, and package benchmark.
- The final `0.2.2` Linux artifacts were rebuilt and verified: client SHA256 `c79a7c0c3b0a1a50b222fa46e26e94154ec05b148c7e382abade843105cd3522`, server SHA256 `01719c0a54d53cefbfcf4421934121b3dd327f72210b39a32704252017bcd09e`.
- Fast two-namespace veth A/B evidence is archived in `docs/benchmark-evidence/v0.2.2/veth/`: TCP throughput median was 107.59 -> 101.25 Mbps (-5.9%), peak server RSS improved 8.95 -> 8.51 MiB (-4.9%), and latency stayed flat to improved on most metrics.

## Known observations
- Live `Broken pipe` and `Connection reset by peer` warnings are still observed and are current work; this release does not claim they are fully fixed.
- The release is intentionally conservative: known rough edges remain, but the core UDP-over-stream, GSO safety, high-pressure, packaging, and benchmark gates are release-ready.
